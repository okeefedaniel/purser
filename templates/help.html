User Manual

Purser is the DockLabs financial-close, submission-review, and compliance product. State agencies and the programs they run use Purser to file periodic financial reports, route them through review and approval, and roll the approved results into a signable close package.

Around the close cycle, Purser also tracks ongoing compliance obligations against grant recipients via an external submitter portal.

A submission moves through these statuses:

  • Draft — being filled in. Editable.
  • Submitted — filed for review. No more submitter edits.
  • Under Review — a reviewer has claimed it.
  • Revision Requested — reviewer asked for changes (comment required). Returns to Draft.
  • Approved — eligible to roll into the close package.
  • Closed — folded into the closed period. Terminal.

Every transition is recorded in the immutable SubmissionStatusHistory audit trail.

Editing requires both the right role and program rostering. A user with purser_submitter can only file submissions for the specific programs they're listed on as a submitter (M2M on the program record). Same for reviewers.

Submissions are also locked once they leave draft or revision_requested. To edit a submitted record, a reviewer has to send it back via Request Revision.

The queue (/purser/review/) lists every submission in submitted or under_review, oldest-first. Click any row to open the review detail.

From there you can Begin Review (claims the submission), Approve, or Request Revision (comment required). Once you claim a submission by approving or requesting revision, only you and admins keep seeing it as personal work; unclaimed items are visible to every reviewer on the program.

A ClosePackage is the per-period consolidated artifact. One per fiscal period. It carries the period's aggregated data, period-over-period and budget variance snapshots, a compliance summary, an executive narrative, and a generated PDF.

Its lifecycle: draft (awaiting submissions) → complete (all approved) → signing (in Manifest) → signed → published.

Two paths:

  • Send to Manifest — when keel.signatures is configured, Purser hands the package off to Manifest with ordered signers. On approval, Manifest sends the signed PDF back; Purser files it as a ClosePackageAttachment and advances the package status automatically.
  • Local sign / upload signed PDF — the fallback when Manifest isn't deployed, or when the package was signed on paper. Upload the signed PDF directly; the package follows the same status path.

The Send to Manifest button is hidden when the integration isn't configured — there are no silent no-ops.

The internal compliance dashboard (/purser/compliance/) shows compliance items generated from keel.compliance templates. You'll see:

  • Overdue items, ordered by due date.
  • Upcoming items in upcoming or pending status.
  • KPIs: active obligations, overdue count, upcoming count, items in submitted or under_review.

Click any item to open its detail view, including its template, obligation, recipient, and full action history.

The portal at /purser/portal/ is the surface for external grant recipients filing compliance documents from outside the agency. Recipients sign in with their DockLabs identity and see only their own agency's compliance items — Purser scopes the result set by the user's linked Agency.

Recipients submit by uploading the requested document and moving the item from pending (or overdue) to submitted.

Purser exposes a per-user inbox endpoint at /api/v1/helm-feed/inbox/. Helm queries it for each user's gating reviews — submissions in submitted or under_review where you're a program reviewer and either nobody has claimed the review yet or you're the claimant.

Items deep-link straight into the review detail page in Purser.

Admin
Full access. Manage programs, run any transition, close periods, send to Manifest, waive compliance items.
Program Officer
Operate programs day-to-day inside the agency. Manage rosters and review assigned programs.
Submitter
File monthly submissions for rostered programs. Edit line values, attach documents.
Reviewer
Review submissions for rostered programs. Begin review, approve, or request revisions.
Compliance Officer
Triage compliance items submitted by external recipients.
Read-Only
View dashboards and signed close packages. No edit.
External Submitter
Grant recipients filing compliance documents from outside the agency. Restricted to their own agency.

Open /notifications/preferences/. For each event type, toggle the in-app and email channels independently.

A few notifications (overdue submissions, overdue compliance, escalations) cannot be muted — they bypass user preferences because they're operationally critical.

Each program can carry a BudgetBaseline for a fiscal year and line item, with an annual amount and an optional monthly spread. The submission form surfaces the monthly slice next to each line so submitters see the budget while typing.

When a line crosses the configured variance threshold, a purser_variance_alert notification fires to reviewers and admins.
